ClipShot

A CleanShot-X-class screenshot tool for Linux & Wayland — built from scratch for the GNOME desktop. Press a key, drag a box on a frozen screen, and the image is already on your clipboard.

Python · GTK4 + libadwaita Wayland-native (GNOME 50) repo · florinhrib/clipshot deploys to any Linux box

01 What it is

ClipShot is your own screenshot app — the Linux answer to CleanShot X on the Mac. It lives in the top-bar tray (no window cluttering your desktop) and turns one keypress into a screenshot that's instantly on your clipboard, ready to paste anywhere. It also annotates, pins shots on top of your screen, extracts text with OCR, and keeps a history — everything CleanShot does, on Linux.

You asked for exactly one thing at its heart: “press a key, drag what I want, and have it copied so I can paste it everywhere.” That's the spine of ClipShot. Everything else is built around it.

02 The core flow

This is what happens the instant you press Super + Shift + S:

1
Capture the whole screen
A full-screen still is grabbed through the secure desktop “portal”. Because it's a frozen photo, the picture can't change while you choose — this is the “freeze the screen” trick that makes selection precise.
2
Show the selection overlay
That still is shown full-screen with a dimmed wash. You drag a box; inside the box stays bright. You get a crosshair, a pixel magnifier loupe, and a live W × H readout.
3
Fine-tune the box
Don't redraw if it's slightly off — drag the handles to resize, drag inside to move, or nudge with arrow keys (1px, or 10px with Shift). Enter confirms, Esc cancels.
4
Crop & copy
The region is cropped and placed on the clipboard as a PNG — before any other UI appears. You can Ctrl+V immediately into a browser, chat, editor, anywhere.
5
The floating HUD
A little thumbnail card slides into the corner with buttons: Copy, Save, Annotate, Pin. Ignore it and keep working, or click in to do more. It never steals your focus.

Pipeline in one line:  hotkey → portal capture → frozen still → drag-crop → wl-copy (clipboard) → HUD → optional annotate / pin / OCR / history.

03 Why this design (the Wayland problem)

On modern GNOME (Wayland), apps are not allowed to freely grab the screen or draw their own overlay on top of everything — that's a deliberate security wall. It's exactly why tools like Flameshot, grim and slurp are broken or flaky on GNOME. ClipShot sidesteps the wall with the only reliable pattern:

  • Capture first, select second. Grab a full-screen still through the official portal, then let you crop it inside a normal app window. No forbidden “live overlay” needed — and it gives the frozen-screen feel for free.
  • Drive the clipboard ourselves. The PNG is handed to wl-copy in its own detached session, so the copied image survives even after every ClipShot window closes — solving the classic Wayland “clipboard empties when the app exits” trap.
  • A real tray icon via a pure D-Bus StatusNotifierItem (rendered by your AppIndicator extension), with zero GTK3 baggage.
Verified on your machine: full-screen capture (4096×2160) → crop → clipboard round-trips byte-for-byte, and the cropped image pastes into any app. 43 automated tests pass.

04 Everything it does

Region capture

Crosshair, pixel magnifier, live dimensions, adjustable handles, arrow-key nudging, aspect-lock.

Fullscreen & window

Grab the whole screen instantly, or a single window (window mode shines once the Shell extension is on).

Auto clipboard

Image on the clipboard the moment you release — paste anywhere, no extra click.

Floating HUD

Corner thumbnail with Copy / Save / Annotate / Pin. Non-blocking; keep working while it waits.

Annotation editor

Arrows, lines, boxes, ellipses, text, highlighter, numbered step-counter, blur & pixelate redaction, crop. Undo/redo.

Pin to screen

Float a shot on top of everything as a reference — resize, adjust opacity, multiple at once.

OCR → text

Drag over any text on screen; the recognized text lands on your clipboard (needs tesseract).

History

A grid of recent captures you can re-copy, re-pin, annotate, or delete.

Self-timer

Delay the capture a few seconds to set up menus or hover states first.

Settings & tray

A full preferences window (every option below) reachable from the tray menu — no terminal needed.

05 Default shortcuts

ShortcutAction
Super + Shift + SCapture a region (the main one)
Super + Shift + FCapture the whole screen
Super + Shift + WCapture a window
Super + Shift + TExtract text (OCR) from a region
Super + Shift + RRepeat the last region

All of these are editable in Settings → Shortcuts. You also chose to bind a mouse side-button to region capture — that's wired through input-remapper (one of the setup steps below). Inside the selector: Enter confirm, Esc cancel, arrows nudge, drag handles to resize.

06 How it's built

It's a single Python package using GTK4 + libadwaita — the same toolkit GNOME itself uses, so it looks and feels native, with no heavy runtime and no compile step to hack on it.

PieceJob
capture.pyGrab the screen via the XDG portal (or the optional Shell extension)
region_selector.pyThe full-screen overlay: crosshair, magnifier, dimensions, handles
clipboard.pyDetached wl-copy / xclip — image & text, persists after exit
app.pyThe tray daemon & brain — one instance, routes every hotkey
hud.py · annotate/ · pin.pyPost-capture HUD, annotation editor, pin-to-screen
ocr.py · history.py · settings_ui.pyText extraction, capture history, the preferences window
tray.py · shortcuts.pyTop-bar icon (D-Bus), GNOME hotkey registration
extension/Optional GNOME Shell extension — sharper window capture & overlay

07 Installing it anywhere

On this machine it's already installed, running, and set to auto-start at login. For any new Linux box (Fedora, Debian/Ubuntu or Arch), one command does everything — installs the app, registers the hotkeys, enables auto-start, and drops in the extension:

# on a fresh machine
curl -fsSL https://raw.githubusercontent.com/florinhrib/clipshot/main/install.sh | bash

To update later: cd ~/clipshot && git pull. To remove: ./uninstall.sh.

08 Three steps to finish on this machine

The core works now. Three optional extras need a one-time action from you (they need your password or a logout, which I can't do for you):

1 · OCR — install the text engine:
sudo dnf install tesseract tesseract-langpack-eng
2 · Mouse side-button — install the remapper, then import the preset:
sudo dnf install input-remapper  → open Input Remapper, import ~/clipshot/presets/input-remapper/clipshot.json, and Apply.
3 · Sharper overlay — log out and back in, then the ClipShot Shell extension loads (and the tray icon settles in). Enable it with gnome-extensions enable clipshot@florinlab.uk if needed.

None of these block daily use — region capture, clipboard, annotation, pin and history all work without them.

09 If something's off

  • No tray icon? The AppIndicator Support GNOME extension must be enabled (it already is on your box). A logout makes it pick up new icons.
  • First capture shows a permission popup? That's GNOME asking to allow screenshots — click Share once; it's silent forever after.
  • Restart the background app: systemctl --user restart clipshot.service
  • See what it's doing: journalctl --user -u clipshot.service -f
  • Change anything: the tray menu → Settings, or edit ~/.config/clipshot/config.json.
